  <action>
Replace the placeholder EvaluateResultPage.tsx with a full implementation. Follow ResultPage.tsx design patterns (Tailwind classes, lab-card, spacing, typography).

**State and data loading:**
- On mount, read `getEvaluateState()` to get `result: EvaluateResult`
- If no result, navigate to `/`
- `activeTab` state: 'ranking' | 'persona' | 'pairwise' (default: 'ranking')

**Header section:**
- Title: "深度评审结果"
- Summary text from `result.summary` (first 200 chars as subtitle)
- "新评审" button: calls `clearEvaluateState()` then `navigate('/')`
- If `result.scoreboard?.too_close_to_call` is true, show a warning badge "差距微小，建议慎重"

**Tab navigation (per CONTEXT.md decision):**
Three tabs styled as horizontal buttons with active state highlighting:
- "综合排名" (ranking) — default
- "评审团详情" (persona)
- "两两对比" (pairwise)

**Tab 1: Overall Ranking (ranking):**
1. **Ranked cards:** For each `result.rankings` entry (sorted by rank), show a `lab-card`:
   - Rank number badge (same style as ResultPage rank badges)
   - Campaign name
   - Verdict badge: ship = green "上线", revise = amber "修改", kill = red "淘汰"
   - Composite score (large number)
   - Panel average score
   - Pairwise record: "W wins / L losses"
   - Top strengths as green tags, top objections as red tags
   - If `result.scoreboard` exists, show verdict_rationale text

2. **BT Score bar chart (per CONTEXT.md decision):**
   If `result.scoreboard` exists, render a simple horizontal bar chart using divs:
   - For each campaign in scoreboard.campaigns (sorted by overall_score desc):
     - Campaign name label
     - Horizontal bar: width = `(score / maxScore) * 100%`, bg-accent
     - Score number at end of bar
   - Use dimension_scores from scoreboard to show a small grid of dimension scores per campaign

**Tab 2: Persona Details (persona):**
- Group `result.panel_scores` by `persona_name`
- For each persona group, show a section header with persona name
- **Persona score cards (per CONTEXT.md decision):** For each score in the group:
  - Avatar icon: use lucide-react `User` icon in a colored circle (cycle through 5 muted colors by persona index)
  - Persona name (bold)
  - Campaign name
  - Score: large number `/10`
  - Reasoning: first 100 chars with expand toggle
  - Strengths as green text items
  - Objections as red text items

**Tab 3: Pairwise Comparison (pairwise):**
1. **Win/loss matrix table (per CONTEXT.md decision):**
   - Build a matrix: rows and columns = unique campaign names from rankings
   - Cell (row=A, col=B): find pairwise_result where campaign_a_id=A & campaign_b_id=B (or reverse)
     - If A won: green "胜" cell
     - If B won: red "负" cell
     - If tie: gray "平" cell
     - Diagonal: dark gray "-"
   - **Position swap inconsistency (per CONTEXT.md decision):** If `position_swap_consistent === false`, show a small warning icon (AlertTriangle from lucide-react, text-amber-500) in the cell with a `title` tooltip: "正反序评审结果不一致"

2. **Pairwise detail cards:** Below the matrix, for each pairwise_result:
   - Card showing: campaign_a_name vs campaign_b_name
   - Winner highlighted
   - Dimensions breakdown: each dimension key with its winner value
   - If not position_swap_consistent: show amber warning bar "位置互换后评审结果不一致" with AlertTriangle icon

**Footer section:**
- Assumptions list (from result.assumptions)
- Confidence notes (from result.confidence_notes)

**Styling rules:**
- Use same Tailwind classes as ResultPage: lab-card, font-display, font-mono, text-muted-foreground, etc.
- Chinese for all user-facing text
- Use lucide-react icons only (User, Trophy, AlertTriangle, ChevronDown, ChevronUp, etc.)
- No additional npm dependencies
