  <action>
1. In `backend/app/models/task.py`, create a helper method and apply WAL+busy_timeout to every sqlite3.connect call:

   Add a private method to TaskManager:
   ```python
   def _connect(self) -> sqlite3.Connection:
       conn = sqlite3.connect(self._db_path(), timeout=10)
       conn.execute("PRAGMA journal_mode=WAL")
       conn.execute("PRAGMA busy_timeout=5000")
       return conn
   ```

   Replace all 5 occurrences of `sqlite3.connect(self._db_path())` (lines 91, 116, 157, 186, 233) with `self._connect()`. The one at line 233 also sets `conn.row_factory = sqlite3.Row` — keep that AFTER `self._connect()`:
   ```python
   conn = self._connect()
   conn.row_factory = sqlite3.Row
   ```

2. In `backend/app/services/brandiction_store.py`, update the existing `_connect()` method (line 251-255):
   ```python
   def _connect(self) -> sqlite3.Connection:
       conn = sqlite3.connect(self._db_path, timeout=10)
       conn.row_factory = sqlite3.Row
       conn.execute("PRAGMA journal_mode=WAL")
       conn.execute("PRAGMA busy_timeout=5000")
       conn.execute("PRAGMA foreign_keys = ON")
       return conn
   ```

3. Also update the health check in `backend/app/__init__.py` (line 84) to use timeout=10:
   ```python
   conn = _sqlite3.connect(db_path, timeout=10)
   ```
   (Health check is a read-only SELECT 1, WAL pragma is optional here, but timeout matters.)

Values chosen:
- `timeout=10` in sqlite3.connect = Python-level wait for lock (seconds)
- `busy_timeout=5000` = SQLite-level retry on SQLITE_BUSY (milliseconds)
- WAL mode allows concurrent readers + one writer without blocking
