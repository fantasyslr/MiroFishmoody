    - Test 1: evaluate_pair calls judge_pair twice per judge (A,B then B,A), producing 6 total votes for 3 judges
    - Test 2: When both orderings agree on winner, position_swap_consistent=True in PairwiseResult
    - Test 3: When orderings disagree (A wins in A-vs-B but B wins when swapped), position_swap_consistent=False

  <action>
    1. Add `position_swap_consistent: bool = True` field to PairwiseResult dataclass in evaluation.py. Also add `swap_votes: List[Dict[str, Any]] = field(default_factory=list)` to store the reversed-order votes separately. Update `to_dict()` in EvaluationResult to include these new fields in pairwise_results serialization.

    2. In pairwise_judge.py `evaluate_pair()`:
       - After getting votes[] from the 3 judges in normal order (A, B), run a second round with arguments swapped: call `self._safe_judge(b, a, judge)` for each judge.
       - For swapped results, the returned "winner" field must be flipped: if swapped judge says "A" it means original B won (because A and B labels are swapped). Flip: "A"->"B", "B"->"A", "tie"->"tie". Same for dimensions.
       - Determine majority winner from normal votes (a_wins_normal vs b_wins_normal) and from swapped votes (a_wins_swap vs b_wins_swap).
       - Set `position_swap_consistent = (normal_majority_winner == swap_majority_winner)`.
       - Final winner_id: use normal majority winner (the swap is for detecting inconsistency, not changing the outcome). If inconsistent and you want to be conservative, keep normal result but flag it.
       - Store swap votes in `swap_votes` field of PairwiseResult.

    3. In `evaluate_all()`, the existing ThreadPoolExecutor pattern remains the same -- each `evaluate_pair` call internally does 2 rounds. No change to evaluate_all's API signature. Bradley-Terry receives the same PairwiseResult list (winner_id unchanged).

    4. Write tests in backend/tests/test_pairwise_debiasing.py:
       - Mock LLMClient to return controlled judge results.
       - Test consistent case: both orderings produce same winner -> position_swap_consistent=True.
       - Test inconsistent case: orderings disagree -> position_swap_consistent=False.
       - Test that winner flipping logic is correct for swapped round.
       - Run with: `cd backend && uv run pytest tests/test_pairwise_debiasing.py -x -v`
  </action>
