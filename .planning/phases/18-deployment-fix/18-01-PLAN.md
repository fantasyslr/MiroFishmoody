<objective>
修复生产入口返回 404 的根因，并建立构建期和运行期的双重防线。

目前 `__init__.py` 在 `dist` 目录不存在时整块跳过静态路由注册，导致 `/` 无任何路由匹配 → 404。同时 `/health` 路由注册在非 API 前缀下，不符合 Railway 健康检查约定。

Purpose: 让生产部署的 `/` 路径一定有路由（200 或 503，不再是 404），让 Railway 可以通过 `/api/health` 轮询服务状态，让构建阶段能提前拦截空白 dist 产物。

Output:
- `backend/app/__init__.py` — 503 fallback + /api/health 路由 + startup log
- `Dockerfile` — 构建断言 `RUN test -f /app/frontend/dist/index.html`
</objective>
