Implementation:
1. Scan all JSON files in `_RESULTS_DIR` directory
2. For each file, load the JSON and extract:
   - `set_id` from the result
   - `created_at` or file modification time as timestamp
   - `scoreboard.campaigns` array — for each campaign: `campaign_name`, `overall_score`
   - `category` from the result metadata (if stored), or infer from campaigns' product_line field. If the result dict has a key like `metadata.category` or the campaigns have product_line, use that. If not determinable, treat as `all`.
3. Filter by `category` param if not `all`:
   - Check if the result's category metadata matches
   - Or check if any campaign's product_line matches the filter value
4. Build response:
```json
{
  "data_points": [
    {
      "set_id": "...",
      "timestamp": "2026-03-17T10:30:00",
      "campaigns": {
        "Campaign A": 7.5,
        "Campaign B": 6.2
      }
    }
  ],
  "campaign_names": ["Campaign A", "Campaign B", ...],
  "category_filter": "all"
}
```
5. Sort `data_points` by timestamp ascending
6. `campaign_names` is the union of all campaign names across all data points (for LineChart legend)
7. Add `@login_required` decorator
8. Handle empty results dir gracefully (return empty data_points array)

To get timestamp: prefer `created_at` field in result JSON. If not present, use file mtime via `os.path.getmtime()`, format as ISO string.

For category inference: check if the result JSON has a top-level `category` key (set during evaluate submission). Also check `metadata.category` from task metadata. If not found, check each campaign's `product_line` field in the scoreboard or in the raw campaign data within the result.
